Exercise 2.3 - Dynamically Loaded Content

Assignment:
(Dynamically loaded content.) As mentioned in the lecture, there are several ways to load content dynamically from JavaScript programs:
1) using the XMLHttpRequest object,
2) using dynamically generated 'iframe' elements,
3) using dynamically generated 'script' tags (see JSONP).
Discuss pros and cons of these three techniques.

2.3.1 XMLHttpRequest

Pros

It has all the good principles from AJAX. AJAX is a combination of technologies to load dynamic content. The interactivity between elements gets delegated in a nice readable way, and content that takes a while to read can be loaded in at any time.

Cons

Handling the output object might differ on the browser, so you have to provide extra code to support eg. Microsofts older versions by using ActiveX. There has to be a way to reexecute the code as it is based on Javascript. Buttons that start a new requests are a way to bypass it.

2.3.2 Iframe

Pros

Used by eg. YouTube to allow developers to easily embed videos on their site. The frame can be created with specified dimensions and a set of properties. Most of the time you do not have to think too much on the output, as the frame is loaded with content from the source's site.

Cons

Back in the day iframes were poorly supported by browsers that are not Firefox, Safari or Explorer. Many firewalls protect against iframes as they have been ways of loading virus and other malware in the past. Extra code has to be added to support these browsers. Content stored in iframes might load slower than the main site, which can cause output issues.

2.3.3 Script tags

Pros

We saw first-hand in Exercise 1.3 that this can be used to load content, when other options are blocked for security reasons.

Cons

Like the cons of XMLHttpRequest we have to reexecute it. Furthermore any Javascript that set up eg. listeners might have to be added again to support the new code. Loading a script from another developers site will give them access to your site yielding security ricks.
