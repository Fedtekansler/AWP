Exercises 2.6 - Code Style

We have run our code through the specified linters. Here are some observations.

JSLint reported "Use spaces instead of tabs" multiple times. It was only warnings, and it was primarily related to lines with comments or indented lines (which makes it a lot).
JSHint reported warnings like the following "Use '!==' to compare with 'null'." and reported unused functions (these are called by HTML but we only added the Javascript).
Google's Closure Linter reported multiple "Illegal tab in comment" and "Missing semicolon after function assigned to a variable". On one file we got 107 errors. That is a lot for code that seems to work perfectly.
ESLint had a lot of nice configuration making it possible to get rid of a lot of the indentation issues shown with the other linters. The only type of errors we saw here was "Parsing error: Unexpected token <".
